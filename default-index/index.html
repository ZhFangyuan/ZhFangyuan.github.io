生活的主线

发表于 2022-04-23 分类于 阅读

最近在阅读鲁迅的一些文章，有些文章感觉很晦涩难懂，于是在知乎上搜索了一下鲁迅。也没看到什么有意思的回答，然后看到知乎的热门话题《哪一刻你意识到，阅读这件事依然无可替代？》，我最近不是正在阅读吗，于是点进去看了下，看到如下的一位答主的回答和该回答下的一条回复，我觉得很有道理。现摘录在这里。

这个话题我挺有感触。

当人毕业几年，到了三十这个年纪的时候，你会有很多压力需要应付，会有很多不开心的事，并且，你还可能会发现，自己可能除了忙工作，不知道还有什么其他事情值得自己投入精力。

当你到了五六十岁，刚退休在家，习惯了日复一日的生活，突然闲了下来，也可能会觉得茫然。

就在前几天我还和我妈探讨过这个问题。我说您现在还年轻，现在退休了要不要找出一块时间来读一些东西，学点以前没有机会学的东西，或者搞个什么业余爱好，至少是一个长远的计划，不要每天就看小红书。

老婆说我对妈要求太多了，每天就乐乐呵呵的不好吗？ 我觉得乐乐呵呵当然好，但是人生不可以能一直一帆风顺，也不可能对同样的生活一直保有热情。

人总需要一个主线剧情，可以不断给你新的风景和新的挑战。让你在困难低谷时，仍然知道自己是为什么乐观地活着。

而阅读和创作就是一个很好的载体。

当然，它不是唯一的载体。企业家可以为了公司拼搏到七八十岁，科学家可以研究一辈子自己喜欢的东西，而画家、作家、音乐家等等都可以自然找到愿意伴随一生的主线剧情。

而对于普通人来说，我们至少可以成为音乐爱好者、摄影爱好者、体育爱好者、咖啡爱好者。或者更入世一点，当一个痴迷于深度了解经济、金融、政治、军事、科技等等任何一个领域的真相爱好者（我新发明的）。

可能是十年象牙塔的生活重新塑造了我的价值观，在我的眼里，知识和真相本身就是有价值的。愿大家都可以找到自己的主线剧情。远离烦恼，快乐生活。

下面是其中一条回复

现实生活中有太多的苦，但这都不能成为人生的全部，说逃避也好，说看开也罢，我们可以根据自己的兴趣和能力，找到一个既能实现又能沉溺进去的事情去做，这就是幸福。人的精力十分有限，**如果被各种压力推着走却不去抓主线的话就会活在别人的世界里，又累又没有自我。所以答主的观点我很赞同，也是我一直劝我爱人的话，就是找到自己感兴趣的喜欢投入时间去做的事，甚至可以把工作当成副本，只当是挣钱养活自己的副本，这样我们的精力就会集中，外界的各种诱惑都基本不存在了，烦恼自然也就少了。我说说自己吧，学到博士三十多岁开始工作，慢慢的对工作的热情也没了，基本属于躺平，不用带脑子都能工作，然后列出自己的兴趣、能力，慢慢找到近期感兴趣的事，只要有空闲就投入其中，工作真的当成了副业了，这种感觉真的很好，好像回到了中学时代，做出一道困难的数学题都能开心半天的时代，现在也基本是这样的状态。不刷朋友圈，不刷各种app，有时只看看知乎，说说自己的想法，近况。引大刘的一段话 “其实，儿子，过一个美妙的人生并不难，听爸爸教你：你选一个公认的世界难题，最好是只用一张纸和一只铅笔的数学难题，比如歌德巴赫猜想或费尔马大定理什么的，或连纸笔都不要的纯自然哲学难题，比如宇宙的本源之类，投入全部身心钻研，只问耕耘不问收获，不知不觉的专注中，一辈子也就过去了。人们常说的寄托，也就是这么回事。**或是相反，把挣钱作为惟一的目标，所有的时间都想着怎么挣，也不问挣来干什么用，到死的时候像葛朗台一样抱者一堆金币说：啊，真暖和啊……所以，美妙人生的关键在于你能迷上什么东西。”<刘慈欣 球状闪电>

其实，我在前面一段时间也在思考，我生活工作的意义在哪里，到底什么才是价值。以前的目标是是考上好的高中，考上心仪的大学，考上研究生，找到一份合适的工作，现在的目标是赚很多钱，未来可能又有什么其他什么目标，但是总觉得缺了点什么。是呀，似乎太多时候，我或者说，我们都是在外界的推动下前进，不知不觉就荒废了太多的时间，还过得很累。生活应该是有“主线”任务的，按照自己的“主线”走，生活的价值才是最大的。

鲁迅《弟兄》阅读笔记

发表于 2022-04-09 分类于 阅读

今天是2022年4月9日。

碰巧阅读了鲁迅先生的文章《弟兄》。文章前面对比了秦益堂的几个儿子和张沛君与他弟弟张靖甫之间的不同。秦益堂的几个儿子常常为金钱大打出手，而张沛君兄弟俩兄弟情深。周围的人对张沛君兄弟俩的关系赞不绝口。

故事的转折。靖甫一天生了病，当时社会上正流行猩红热，张沛君请的中医大夫也诊断为猩红热。此时的张沛君心里幽生了许多非常实际的想法。弟弟去世后的安葬的费用，如何照顾安排弟弟的子女，首先当然是安排自己的子女，弟弟的子女就顾不上了。但是又担心周围人对自己的看法。

当然，以上都是张沛君自己心里想的，好在西医看了之后，确认不是猩红热，只是普通的疹子，张沛君才松了一口气。

平时相处融洽的两兄弟，在真正遇到困难的时候，似乎也会因为各自的考量而变得自私自己。有人说鲁迅在讽刺这样一种虚伪的人。我在想，在物资匮乏，自身也难保的情况下，心里有这样的龌龊想法，似乎也是人之常情。

最近上海因为疫情的原因，大家都被关在家里，物资紧张。我一直在公司值守，最近隔离结束，需要回家隔离了。我给同住的室友打了电话，说我这两天要回去了。他们却在因为多一个人吃饭而担心不够吃了。我们平时相处也还算融洽，生活物资的确短缺，但倒不至于到达让人饿肚子的境地。但是却已经有人在为这样的事情计较了。

社会发展到现在，大多数人都被教化得有礼的。我们会自觉排队，遇到红灯会停下来，捐赠物资给有需要的人，我们会关心身边的小猫小狗、花花草草。但是当这个世界的秩序被打乱的时候，我们真的还能保持人性中美好的东西么，还是暴露出兽性邪恶的一面。

《弟兄》里张沛君的心理活动，我觉得是一种十分微妙的感觉，展现的是一个人的矛盾，不能说是对这个人的讽刺，这是人真实的一种状态。这种状态真实，但是我又觉得很不舒服。

我不知道鲁迅写这篇文章的时候是否也是抱有一种十分矛盾的心情，充满了对人性的无奈。

不修改数组找出重复的数字

发表于 2022-03-19 分类于 算法

题目描述

在一个长度为n+1的数组里的所有数字都在1~n的范围内，所以数组中至少有一个数字是重复的。请找出数组中任意一个重复的数字，但不能修改输入的数组。例如，如果输入长度为8的数组{2，3，5，4，3，2，6，7}，那么对应的输出是重复的数字2或者3。

解题：数组的长度为n+1，数字的范围却只在1~n。对于任意一个数字i，我们把它放在数组中索引为（i-1）的位置。当数字i有多个的时候，将重复的数字i放在索引为（i-1）的位置的时候，就会冲突，如此找出重复的数字。比较关键的一点是，数组的长度大于最大的数字的大小，意味着，数组的最后一个位置不会有相应的数字放置，因此可以从数组末尾开始找重复的数字，一直向前推进。

java单例模式

发表于 2022-03-01 分类于 算法

1、实例为私有对象，不可随意访问，因此一种显而易见的方式是通过get方法获取实例；
2、get方法各个线程都可以访问，并创建实例，此时会生成多个实例，不满足单例模式的要求；
3、一个显而易见的思路是，当一个线程创建实例的时候，需要判断一下该实例是否为null，不为null可直接返回现有的实例对象；
4、如果为null的话，说明当前没有线程创建该实例，因此需要对该实例进行初始化，且这种初始化只能进行一次，且只能有一个线程做这个操作；
5、因此，在某个线程发现实例为null的时候，加锁，进行创建实例的操作，在锁里面面，要先判断一下该实例是否为null，不为null才初始化实例；
6、以上为double check的基本思路，但是此时还是不能保证是单例模式，因为java底层会进行指令重排，因此需要对私有对象instance进行volatile关键字限制

饿汉式

饿汉式是线程安全的，但不是lazy加载。

双检测

目的是提高访问效率，如果没有最外层的判空语句，那么每次调用getInstance()方法，都会互斥访问，影响了访问效率，而加了双重判断效率大大提高。注意，这里还用到了volatile关键字，目的是为了防止指令重排，假设没有这个关键字的话，当一个进程在创建单例对象的时候，假设只创建了一半，此时单例对象已经不为null了，其他线程就会得到一个不完整的单例对象。

同步枷锁

直接将方法给锁住，实现简单，但是开销大。

pulsar调研

pulsar调研

1、pulsar单机部署

pulsar可以单机部署，单机部署包含一个pulsar broker，zookeeper和bookzeeper（中文意义：簿记员，记账员）。

注意：pulsar默认需要2G的jvm堆内存，所以启动的时候可能会因为堆内存不够而无法启动，可以前往conf/pulsar_env.sh修改PULSAR_MEM这个参数。

单机部署的时候，可以选择安装builtin connectors 和tiered storage offloaders ，这两个组件的具体作用后面再补充。这两个组件不是必须要安装的。

启动pulsar后。可以生产和消费消息。需要注意的是，pulsar可以不用生产消息也可以消费消息，这句话的含义是，当你启动一个consumer去订阅一个topic并且消费消息的时候，如果provider并没有这个topic和相关的消息，生产者会主动创建一个topic，并产生消息，供消费者消费。

  • 保证消息到达 Apache BookKeeper ，持久化的消息存储
  •  Pulsar Functions 轻量级计算框架，数据流处理
  •  Pulsar IO 处理数据的流入流出pulsar
  • Tiered Storage 数据冷热数据转移

message

生产者生产消息，消费者消费消息，消费者没消费一条消息，会向broker传递一个ack，表示该条消息已被消费。pulsar默认会保留所有的消息，只有当消费者将消息全部消费之后，pulsar才会删除这些消息。如果消费者消费消息失败，可以重试消费。

元素 描述
value/data payload 消息值
key 每条消息可以设置一个key值，可选
properties key/value形式的map，用户自定义的属性map
producer name 生产者名字，可选，不设定则为默认名
sequence id 每一条消息都属于某个topic的一个有序的序列，sequence id表示该消息的序号
publish time 消息产生的时间，由生产者自动提供
event time 应用标注的时间，个人理解为消息被消费的时间戳
TypedMessageBuilder 构造消息，具体用处后面再研究

消息的默认大小为5M，也自定义大小。

  • 在broker.conf file.

  • 在bookkeeper.conf file.

Producers

producer生产某个topic的消息给broker，broker处理这些消息。

消息发送模式分为异步和同步两种方式。

模式 描述
同步 生产者将消息发送给broker之后，会同步等待broker返回一个ack，若未收到ack，则认为该消息发送失败。
异步 生产者将消息发送给一个队列，然后立即返回，消息队列后台处理。若消息队列满了，则直接返回失败或者消息堵塞。消息队列大小可自定义。

接入模式（access mode）

对于生产者有多种接入模式

接入模式 描述
shared 多个生产者可以将消息发送到同一个topic，这是默认模式
exclusive 只能一个生产者发布消息到某个topic，即独占该topic。其他生产者发布消息到该topic将会报错。当old生产者经历完一个网络分片之后将会被驱逐，然后选举一个新的生产者独占该topic。选举方式后续研究。
WaitForExclusive 当一个topic被某个生产者独占的时候，新的生产者将等待一个pending时间，而不是直接报错，时间到了，选择一个producer独占该topic。

消息压缩

支持以下类型

  • LZ4
  • ZLIB
  • ZSTD
  • SNAPPY

Batching

消息可以一个批次一个批次地发送，提高发送的效率。

chunking

当消息过大，可以分块发送，消费者收到这些chunk之后再组装成一条完整的消息。
