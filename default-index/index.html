需要注意收入和利得、费用和损失的区分：
①利得是指由企业非日常活动所形成的、会导致所有者权益增加的、与所有者投入资本无关的经济利益的流入。从概念上可以看出，收入属于日常活动，而利得属于非日常活动。例如，企业将产品对外出售（日常活动），此业务属于收入，而企业接受非关联方的现金捐赠（非日常活动），此业务属于利得；

②损失是指由企业非日常活动所发生的、会导致所有者权益减少的、与向所有者分配利润无关的经济利益的流出。从概念上同样可以看出，费用属于日常活动，而损失属于菲日常活动。例如，企业支付管理人员薪酬（日常活动），此业务属于费用，而企业支付税收罚款（非日常活动），此业务属于损失。

第三章：存货

考点1：存活的确认和初始计量（**）

存货，是指企业在日常活动中持有以备出售的产成品或商品、处在生产过程中的在产品、在生产过程或提供劳务过程中耗用的材料和物料等。包括原材料、库存商品、周转材料、发出商品、委托加工物资、在途物资、在产品、产成品等。

存货的持有目的是为了耗用或出售，注意与工程物资的区分，工程物资是为了购建符合资本化条件固定资产而购入的，不属于存货。

两种考试方式：1、存货的入账价值；2、哪些应当构成取得存货的成本

在成品：生产工序未完成

半成品：生产工序已完成

存货的初始计量

企业取得存货应当按照成本进行计量。

1.外购的存货入账成本=卖价+相关税费+运费+装卸费+保险费+途中合理损耗+入库前挑选整理费等

2.采购过程中的损耗处理
(1）从供货单位、外部运输机构等收回的物资短缺或其他赔款，冲减存货的采购成本；
(2）因遭受意外灾害发生的损失和尚待查明原因的途中损耗，不得增加存货的采购成本，应暂作为待处理财产损溢进行核算，查明原因后再作处理；

3.其他方式取得的存货
(1）投资者投入的存货按合同或协议约定的价值入账，但合同或协议约定价值不公允的除外；
(2）盘盈存货成本按其重置成本作为入账价值，按管理杈限批准后冲减“管理费用〞
(3）通过提供劳务取得的存货成本=直接人工＋其他直接费用＋归属于该存货的间接费用。

4.特殊方式存货成本确定
1）非正常消耗的直接材料、直接人工和制造费用，应在发生时计入当期损益，不应计入存货成本。如由于自然灾害而发生的直接材料、直接人工和制造费用，由于这些费用的发生无助于使该存货达到目前场所和状态，不应计入存货成本，而应确认为当期损益。
(2） 仓储费用指企业在存货采购入库后发生的储存费用，应在发生时计入当期损益。但是，在生产过程中为达到下一个生产阶段所必需的仓储费用应计入存货成本。

(3）不能归属于使存货达到目前场所和状态的其他支出，应在发生时计入当期损益，不得计入存货成本。
(4）企业采购用于广告营销活动的特定商品，向客户预付货款末取得商品时，应作为预付账款进行会计处理，待取得相关商品时计入销售费用。

redis scan命令查询为空问题分析及解决

1、问题背景

我们知道，在生产中，不能使用keys命令来查找key，因为redis是单线程的，当数据库中的key很多的时候，由于keys命令是一次性遍历数据库中所有的key，这会导致redis被阻塞。在redis的2.8及以后的版本中，为了能够优化查找key的过程，增加了scan命令，scan可以通过迭代的方式扫描数据库中的key，也就是说一次只查询部分数据，这样就可以避免因为一次查询过多数据导致redis阻塞的问题。

正好，一次我们需要在生产上查找key，因此使用了如下的命令。

我们期待从数据库中查找匹配0200开头的key，一次查找10个，但是结果返回的却为空。这是怎么回事儿呢？我们接着往下看。

2、原因分析

scan是redis提供的数据查询命令，我们当然是直接查看redis的官方文档（https://redis.io/commands/scan/）就行了。

2.1 scan命令

scan命令使用的基本格式如下

一共有3个参数，第一个参数cursor是游标，该参数具体什么含义后续再介绍。第二个参数是MATCH，即模式匹配，查询符合要求的key。第三个参数是COUNT，这个参数的含义是一次遍历的索引的数量，索引什么含义也在后面介绍。这里需要知道的一点就是，COUNT不是说一次查询count数量的key值。因此我们刚开始期望用scan命令查找10个符合要求的key是不对的。

2.2 字典格式

我们在2.1中卖了个关子，即cursor游标是什么，索引是什么，我们没有说清楚，这是因为这涉及到redis的字典结构，不讲清楚redis的字典结构，我们就没办法说清楚上面两个概念。

redis的字典格式类似于java中的hashmap结构，是数据+链表的结构，示意图如下所示。

cursor游标指的就是数组的索引，但是这个索引不是从0到n这么简单，因为数组会随着key的增加扩容和随着key的减少缩容，因此索引值会有一些变化。然后COUNT指的是每次遍历的是数组的索引数量，例如，假设当前的cursor为0，count值为5，则scan查询为扫描索引0到4的数据（假设这里的cursor是线性增加的）。从这里可以得知，每次调用scan命令的时候，匹配出来的key的数量有可能比COUNT的值大，原因是每个索引下还有一个链表，这个链表里还有可能有符合查询条件的key。

如上图所示，假设数组原来的长度为8，经过扩容变为16（容量一般扩为原来的2倍）。以索引3（0011）为例，key值通过hash之后对数组的长度取模得到对应的索引位置。当扩容之后，索引的位置会变为3（0011）或者11（1011），举几个例子。

扩容之后，索引3（0011）上的key大致会均分在索引3（0011）和11（1011）的位置上。但是按照这样的扩容和缩容，scan在扫描数据的时候会导致数据遍历重复或者遗漏。实际上的扩容和缩容如下图所示。

scan 的遍历顺序非常特别。它不是从第一维数组的第 0 位一直遍历到末尾，而是采用了高位进位加法来遍历。之所以使用这样特殊的方式进行遍历，是考虑到字典的扩容和缩容时避免槽位的遍历重复和遗漏。

高位进位法从左边加，进位往右边移动，同普通加法正好相反。但是最终它们都会遍历所有的槽位并且没有重复。

生活的主线

发表于 2022-04-23 分类于 阅读

最近在阅读鲁迅的一些文章，有些文章感觉很晦涩难懂，于是在知乎上搜索了一下鲁迅。也没看到什么有意思的回答，然后看到知乎的热门话题《哪一刻你意识到，阅读这件事依然无可替代？》，我最近不是正在阅读吗，于是点进去看了下，看到如下的一位答主的回答和该回答下的一条回复，我觉得很有道理。现摘录在这里。

这个话题我挺有感触。

当人毕业几年，到了三十这个年纪的时候，你会有很多压力需要应付，会有很多不开心的事，并且，你还可能会发现，自己可能除了忙工作，不知道还有什么其他事情值得自己投入精力。

当你到了五六十岁，刚退休在家，习惯了日复一日的生活，突然闲了下来，也可能会觉得茫然。

就在前几天我还和我妈探讨过这个问题。我说您现在还年轻，现在退休了要不要找出一块时间来读一些东西，学点以前没有机会学的东西，或者搞个什么业余爱好，至少是一个长远的计划，不要每天就看小红书。

老婆说我对妈要求太多了，每天就乐乐呵呵的不好吗？ 我觉得乐乐呵呵当然好，但是人生不可以能一直一帆风顺，也不可能对同样的生活一直保有热情。

人总需要一个主线剧情，可以不断给你新的风景和新的挑战。让你在困难低谷时，仍然知道自己是为什么乐观地活着。

而阅读和创作就是一个很好的载体。

当然，它不是唯一的载体。企业家可以为了公司拼搏到七八十岁，科学家可以研究一辈子自己喜欢的东西，而画家、作家、音乐家等等都可以自然找到愿意伴随一生的主线剧情。

而对于普通人来说，我们至少可以成为音乐爱好者、摄影爱好者、体育爱好者、咖啡爱好者。或者更入世一点，当一个痴迷于深度了解经济、金融、政治、军事、科技等等任何一个领域的真相爱好者（我新发明的）。

可能是十年象牙塔的生活重新塑造了我的价值观，在我的眼里，知识和真相本身就是有价值的。愿大家都可以找到自己的主线剧情。远离烦恼，快乐生活。

鲁迅《弟兄》阅读笔记

发表于 2022-04-09 分类于 阅读

今天是2022年4月9日。

碰巧阅读了鲁迅先生的文章《弟兄》。文章前面对比了秦益堂的几个儿子和张沛君与他弟弟张靖甫之间的不同。秦益堂的几个儿子常常为金钱大打出手，而张沛君兄弟俩兄弟情深。周围的人对张沛君兄弟俩的关系赞不绝口。

故事的转折。靖甫一天生了病，当时社会上正流行猩红热，张沛君请的中医大夫也诊断为猩红热。此时的张沛君心里幽生了许多非常实际的想法。弟弟去世后的安葬的费用，如何照顾安排弟弟的子女，首先当然是安排自己的子女，弟弟的子女就顾不上了。但是又担心周围人对自己的看法。

当然，以上都是张沛君自己心里想的，好在西医看了之后，确认不是猩红热，只是普通的疹子，张沛君才松了一口气。

平时相处融洽的两兄弟，在真正遇到困难的时候，似乎也会因为各自的考量而变得自私自己。有人说鲁迅在讽刺这样一种虚伪的人。我在想，在物资匮乏，自身也难保的情况下，心里有这样的龌龊想法，似乎也是人之常情。

最近上海因为疫情的原因，大家都被关在家里，物资紧张。我一直在公司值守，最近隔离结束，需要回家隔离了。我给同住的室友打了电话，说我这两天要回去了。他们却在因为多一个人吃饭而担心不够吃了。我们平时相处也还算融洽，生活物资的确短缺，但倒不至于到达让人饿肚子的境地。但是却已经有人在为这样的事情计较了。

社会发展到现在，大多数人都被教化得有礼的。我们会自觉排队，遇到红灯会停下来，捐赠物资给有需要的人，我们会关心身边的小猫小狗、花花草草。但是当这个世界的秩序被打乱的时候，我们真的还能保持人性中美好的东西么，还是暴露出兽性邪恶的一面。

《弟兄》里张沛君的心理活动，我觉得是一种十分微妙的感觉，展现的是一个人的矛盾，不能说是对这个人的讽刺，这是人真实的一种状态。这种状态真实，但是我又觉得很不舒服。

我不知道鲁迅写这篇文章的时候是否也是抱有一种十分矛盾的心情，充满了对人性的无奈。

不修改数组找出重复的数字

发表于 2022-03-19 分类于 算法

题目描述

在一个长度为n+1的数组里的所有数字都在1~n的范围内，所以数组中至少有一个数字是重复的。请找出数组中任意一个重复的数字，但不能修改输入的数组。例如，如果输入长度为8的数组{2，3，5，4，3，2，6，7}，那么对应的输出是重复的数字2或者3。

解题：数组的长度为n+1，数字的范围却只在1~n。对于任意一个数字i，我们把它放在数组中索引为（i-1）的位置。当数字i有多个的时候，将重复的数字i放在索引为（i-1）的位置的时候，就会冲突，如此找出重复的数字。比较关键的一点是，数组的长度大于最大的数字的大小，意味着，数组的最后一个位置不会有相应的数字放置，因此可以从数组末尾开始找重复的数字，一直向前推进。
