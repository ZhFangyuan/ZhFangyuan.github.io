java单例模式

发表于 2022-03-01 分类于 算法

1、实例为私有对象，不可随意访问，因此一种显而易见的方式是通过get方法获取实例；
2、get方法各个线程都可以访问，并创建实例，此时会生成多个实例，不满足单例模式的要求；
3、一个显而易见的思路是，当一个线程创建实例的时候，需要判断一下该实例是否为null，不为null可直接返回现有的实例对象；
4、如果为null的话，说明当前没有线程创建该实例，因此需要对该实例进行初始化，且这种初始化只能进行一次，且只能有一个线程做这个操作；
5、因此，在某个线程发现实例为null的时候，加锁，进行创建实例的操作，在锁里面面，要先判断一下该实例是否为null，不为null才初始化实例；
6、以上为double check的基本思路，但是此时还是不能保证是单例模式，因为java底层会进行指令重排，因此需要对私有对象instance进行volatile关键字限制

饿汉式

饿汉式是线程安全的，但不是lazy加载。

懒汉式

双检测

目的是提高访问效率，如果没有最外层的判空语句，那么每次调用getInstance()方法，都会互斥访问，影响了访问效率，而加了双重判断效率大大提高。注意，这里还用到了volatile关键字，目的是为了防止指令重排，假设没有这个关键字的话，当一个进程在创建单例对象的时候，假设只创建了一半，此时单例对象已经不为null了，其他线程就会得到一个不完整的单例对象。

同步枷锁

直接将方法给锁住，实现简单，但是开销大。

pulsar调研

发表于 2022-01-10 分类于 技术

pulsar调研

1、pulsar单机部署

pulsar可以单机部署，单机部署包含一个pulsar broker，zookeeper和bookzeeper（中文意义：簿记员，记账员）。

注意：pulsar默认需要2G的jvm堆内存，所以启动的时候可能会因为堆内存不够而无法启动，可以前往conf/pulsar_env.sh修改PULSAR_MEM这个参数。

单机部署的时候，可以选择安装builtin connectors 和tiered storage offloaders ，这两个组件的具体作用后面再补充。这两个组件不是必须要安装的。

启动pulsar后。可以生产和消费消息。需要注意的是，pulsar可以不用生产消息也可以消费消息，这句话的含义是，当你启动一个consumer去订阅一个topic并且消费消息的时候，如果provider并没有这个topic和相关的消息，生产者会主动创建一个topic，并产生消息，供消费者消费。

  • 保证消息到达 Apache BookKeeper ，持久化的消息存储
  •  Pulsar Functions 轻量级计算框架，数据流处理
  •  Pulsar IO 处理数据的流入流出pulsar
  • Tiered Storage 数据冷热数据转移

message

生产者生产消息，消费者消费消息，消费者没消费一条消息，会向broker传递一个ack，表示该条消息已被消费。pulsar默认会保留所有的消息，只有当消费者将消息全部消费之后，pulsar才会删除这些消息。如果消费者消费消息失败，可以重试消费。

元素 描述
value/data payload 消息值
key 每条消息可以设置一个key值，可选
properties key/value形式的map，用户自定义的属性map
producer name 生产者名字，可选，不设定则为默认名
sequence id 每一条消息都属于某个topic的一个有序的序列，sequence id表示该消息的序号
publish time 消息产生的时间，由生产者自动提供
event time 应用标注的时间，个人理解为消息被消费的时间戳
TypedMessageBuilder 构造消息，具体用处后面再研究

消息的默认大小为5M，也自定义大小。

  • 在broker.conf file.

  • 在bookkeeper.conf file.

Producers

producer生产某个topic的消息给broker，broker处理这些消息。

消息发送模式分为异步和同步两种方式。

模式 描述
同步 生产者将消息发送给broker之后，会同步等待broker返回一个ack，若未收到ack，则认为该消息发送失败。
异步 生产者将消息发送给一个队列，然后立即返回，消息队列后台处理。若消息队列满了，则直接返回失败或者消息堵塞。消息队列大小可自定义。

接入模式（access mode）

对于生产者有多种接入模式

接入模式 描述
shared 多个生产者可以将消息发送到同一个topic，这是默认模式
exclusive 只能一个生产者发布消息到某个topic，即独占该topic。其他生产者发布消息到该topic将会报错。当old生产者经历完一个网络分片之后将会被驱逐，然后选举一个新的生产者独占该topic。选举方式后续研究。
WaitForExclusive 当一个topic被某个生产者独占的时候，新的生产者将等待一个pending时间，而不是直接报错，时间到了，选择一个producer独占该topic。

消息压缩

支持以下类型

  • LZ4
  • ZLIB
  • ZSTD
  • SNAPPY

Batching

消息可以一个批次一个批次地发送，提高发送的效率。

chunking

当消息过大，可以分块发送，消费者收到这些chunk之后再组装成一条完整的消息。

会计学基础

发表于 2021-12-08 更新于 2022-05-03 分类于 会计学

会计信息形成的三个问题

主体：对谁的经济交易与事项进行记录和报 告

时点：在什么时点对经济交易与事项进行记录和报告

金额：以什么金额对经济交易与事项进行记录和报告

1、会计主体假设

概念：经济活动的会计处理是以经济个体为单位而认定的，即某一企业的活动与业主及其他企业的活动必须完全分开处理。

会计主体==法律主体

会计主体超越法律主体

2、持续经营假设

概念：假定企业将持续存在下去，而不会在可预见的将来清算解散。

3、会计分期假设

概念：将企业的经营时间划分为等长的时段

4、货币计量假设

概念：以货币为计量单位（记账本位币）

谨慎性原则：存在两种可选会计方法时，应当选择低估好处、高估损失的方法。

会计恒等式

资产=负债+所有者权益

（资金的使用结果） （资金的来源：借债或者股东）

借方 贷方

收入-费用=利润

资产

概念：企业过去的交易或者事项形成的，由企业拥有或者控制的，预期会给企业带来经济利益的资源。

现金资产 非现金资产

货币资金 不以现金形式存在的资产

流动资产（企业可以在1年内或者超过1年的一个营业周期内变现或者耗用的资产）

库存现金、银行存款、应收款项（应收票据、应收账款）、预付款项、存货

非流动资产（企业不能在1年或者超过1年的一个营业周期内变现或耗用的资产）

长期股权投资、固定资产、无形资产

隐性资产：它可能是一项经济资源，对企业有价值，但是没有以资产的名义记录到会计系统

品牌、人力资源、商业关系、社会关系

负债和所有者权益

负债：过去的交易、事项形成的现时义务，履行该义务预期会导致经济利益流出企业

流动负债 偿还压力大，偿还期限长

短期借款、应付及预收款项、应付账款、预收账款、应交税金

长期负债 偿还压力大，偿还期限短

长期借款、应付债券、长期应付款

隐性负债：诉讼 、担保、其他（承诺：七天无条件免费退货）

所有者权益：企业投资者对企业净资产的所有权，包括企业所有者的投资及其增加的权益

所有者权益就是企业的投资者对企业的剩余索取权。

所有者直接投入的资本

企业累计经营经营成功

会计恒等式举例1

1月3日，股东认缴企业注册资本500000，存入企业银行账户

资产（银行存款（+500000））=负债+所有者权益（实收资本（+500000））

1月5日，企业从银行存款提取现金5000元备用

资产（现金（+5000）+银行存款（500000-5000））=负债+所有者权益（500000）

1域月6日，企业从供应商得到面粉等原料，总价8000元，钱未付清

资产（原材料（+8000））=负债（应付账款（+8000））+所有者权益

1月7日，企业购买烤箱等设备，以银行存款支付价款30000.设备已经运到。

资产（固定资产（设备+30000）+银行存款（495000-30000））=负债+所有者权益

1月8号，企业以银行存款偿还供应商欠款8000元

资产（银行存款（-8000））=负债（应付账款（-8000））+所有者权益

六大会计要素

权责发生制与收入

什么时候、什么情况下确认收入

收付实现制：以现金的收支作为确认的依据（会计实务中不允许使用）

权责发生制：以权利已经形成或义务已经发生作为确认的依据

收入：企业在日常活动中形成的、会导致所有者权益增加的、与所有者投入资本无关的经济利益的总流入

主营业务收入、其他业务收入、投资收益、营业外收入、公允价值变动受益

收入确认：

收入确认必须符合两项基本条件

收入的赚取过程已经完成、义务已经完成

经济利益很可能流入企业

费用与利润

费用：企业在日常生活中发生的、会导致所有者权益减少的、与向所有者分配利润无关的经历利益的总流出。

收入与费用配比

费用确认：将相关联的收入与费用在同一会计期间入账（包括期间及金额比上的”配比“）

主营业务收入 主营业务成本

其他业务收入 其他业务成本

投资收益 投资损失

营业外收入 营业外支出

500元蛋糕的费用构成

原材料 主营业务成本

快递运输 销售费用

广告费用 销售费用、管理费用、财务费用

有交易的时候：税金及附加

所得税

资产减值损失

利润：企业在一定会计期间内的经营成果，包括收入减去费用后的净额、直接计入当期利润的利得和损失

利润：营业利润+利得、损失

利润总额-所得税费用=净利润

会计恒等式举例2

1月9日，企业以现金购进饮料2000瓶，价格2000元

资产（现金（-2000）+库存商品（+2000））=负债+所有者权益

1月10日，企业卖出饮料100瓶，收到现金200元

资产（现金（+200））=负债+所有者权益（收入（+200））

会计科目

会计科目：相同性质的经济事项归为一类，并明确其核算内容，而这些项目，在会计上就称为会计科目

会计科目的分类

按照反映的经济内容分：资产类科目、所有者权益类科目、损益类科目、负债类科目、成本类科目

按照隶属关系分类：总账科目、明细科目、一级明细、二级明细、三级明细等

资产类科目：流动资产科目、非流动资产科目

负债类科目：流动负债类科目、非流动资产类科目

共同类科目：既有资产性质，又有负债性质

所有者权益类科目：实收资本（注册资本范围内的投入）、资本公积（指投资者或者他人投入到企业、所有权归属于投资者、并且投入金额上超过法定资本部分的资本；包括资本溢价、资产评估增值、捐赠资本和资本折算差额。）、盈余公积（指公司按照规定从净利润中提取的各种累计资金（公司职工福利设施等））、本年利润（企业档期实现的净利润或者净亏损）、利润分配（核算分红、企业留存（未分配利润））

成本类科目：成本类科目是反映成本费用和支出的，用于核算成本的发生和归集情况，提供成本相关会计信息的会计科目。

损益类可科目：收入类科目、费用类科目，期末余额转入本年利润

会计循环

会计凭证：原始凭证->记账凭证

会计分录

财务报表

资产负债表：资产=负债+所有者权益

资产：按照流动性由强到弱排列

负债：按照流动性由强到弱排列（还款压力大，偿还期限短）

利润表：利润=收入-费用

现金流量表：

负债：债务的负债+非债务的负债

债务：需要负担利息的负债

非债务：不需要承担利息的负债

资金筹集：所有者权益

债券筹资：

<1年：流动负债-”交易性金融负债“

>1年：长期负债-”应付债券“

本年利润的核算

利润=收入-费用

修饰符

接口前加上public，则可以被所有类继承；若不显示添加public修饰符，则只能被同一个包中的类继承。

抽象方法

抽象方法可以继承接口，但是可以不用像普通类一样实现这些接口，即可以只实现接口中的部分方法。

接口方法

接口中的方法都是public abstract类型的，因此对于一个接口可以不用显示地写出public和abstract两个修饰符。

常量

接口中不能拥有变量，但是可以拥有常量，且这些常量前地修饰符为public、final、static，这几个修饰符默认不用写。

接口回调

接口回调是指让接口调用实现类的方法。举个例子。

回调指的是Hello hello = new HelloImpl();这一行代码，我们在开发的过程中经常会用到接口回调，我们可以把hello看作一个指针，指向了实例对象HelloImpl，接着就可以操作HelloImpl对象的各个方法了。
